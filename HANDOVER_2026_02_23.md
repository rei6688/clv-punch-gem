| Key pattern | Dữ liệu | TTL |
|---|---|---|
| `punch:config:isEnabled` | `true/false` | ∞ |
| `punch:config:schedule` | `{ "1":"wfh", "2":"wio", ... }` | ∞ |
| `punch:config:telegram` | object | ∞ |
| `punch:config:times` | `{ am, pm }` | ∞ |
| `punch:day:{YYYY-MM-DD}:off` | `true` | 3 ngày |
| `punch:day:{YYYY-MM-DD}:am` | status object | 3 ngày |
| `punch:day:{YYYY-MM-DD}:pm` | status object | 3 ngày |
| `punch:day:{YYYY-MM-DD}:mode_override` | `"wfh"/"wio"` | **7 ngày** ⚠️ |

### Vấn đề với TTL
- `mode_override` expire sau **7 ngày** → không có lịch sử lâu dài
- Không có lịch sử swap dates (ai swap, khi nào, swap gì)
- KV Redis không phù hợp để query/aggregate history

## 🐛 Bug chính đã fix trong session này

### Bug 1: Header luôn hiển thị "VĂN PHÒNG" dù đã swap → WFH
**Root cause:** `renderStatusGuide()` trong `app-enhanced.js` tính `effectiveMode` từ `schedule[dayIdx]` ở client-side — bỏ qua KV override.  
**Fix:** `getFullDayState()` trong `lib/kv.js` giờ tính `effectiveMode` và `scheduleMode` server-side và trả về cùng response. `renderStatusGuide` đọc `day.effectiveMode` trực tiếp.

### Bug 2: Swap modal luôn show VP dù ngày đó đã được swap → WFH
**Root cause:** `showSwapDayModal()` build `baseMode` từ `schedule[dayIdx]` — không đọc KV override. Kết quả: mở modal lại → thấy VP → click → ghi đè WFH bằng VP.  
**Fix:** Modal giờ gọi bulk API `GET /api/state?dates=d1,d2,...` ngay sau khi render, patch `effectiveMode` vào từng ngày từ KV, re-render. Nút toggle dùng `effectiveMode` thay vì `baseMode`.

### Bug 3: Server 500 khi swap today
**Root cause:** `triggerGitHubWorkflow()` throw khi không có `GITHUB_PAT` ở local.  
**Fix:** Wrapped trong `try/catch` — swap KV vẫn xảy ra, GHA trigger bị skip với `console.warn`.

## 🏗️ Persistent DB — IMPLEMENTED (Turso / LibSQL)

### What was done
The migration plan from the previous session has been **completed**:

1. **Turso (LibSQL/SQLite)** chosen for persistent swap storage — free 500 DBs, edge-compatible
2. `@libsql/client` installed
3. `lib/db.js` created with: `saveSwapOverride`, `getSwapOverride`, `getBulkSwapOverrides`, `listSwapHistory`, `savePunchResult`, `isAvailable`
4. `lib/kv.js` updated:
   - `getDayModeOverride()` reads Turso DB first, falls back to KV cache
   - `setDayModeOverride()` writes to KV with 30-day TTL (was 7 days)
5. `api/actions.js` updated: `swapDay` handler persists to Turso DB in addition to KV cache
6. Vercel KV kept for: `isEnabled`, `schedule`, `telegram`, `times` (config keys, no history needed)

### Schema (auto-created on first request)
```sql
CREATE TABLE swap_overrides (
  date TEXT PRIMARY KEY,
  mode TEXT NOT NULL CHECK(mode IN ('wfh','wio')),
  created_at INTEGER NOT NULL,
  expires_at INTEGER
);
CREATE TABLE punch_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  date TEXT NOT NULL,
  period TEXT NOT NULL CHECK(period IN ('am','pm')),
  status TEXT NOT NULL,
  punch_time TEXT,
  created_at INTEGER NOT NULL
);
```

### New env vars needed
```env
TURSO_DATABASE_URL=libsql://your-db-name.turso.io
TURSO_AUTH_TOKEN=your-auth-token
```

### Graceful fallback
If `TURSO_DATABASE_URL` is not set, `lib/db.js` returns null/empty from all functions.
`getDayModeOverride` in `kv.js` catches DB errors and falls back to KV.
`swapDay` in `actions.js` catches DB write errors and logs a warning.
No breaking changes to the API contract.

## ⚠️ Known limitations / cần theo dõi

1. **`mode_override` KV TTL 30 days** — KV is now a cache layer. Turso DB is the persistent source of truth. If Turso is not configured, KV still works standalone with a 30-day TTL (was 7 days).

2. **Weekend days trong modal** — T7/CN với `baseMode='off'` nhưng đã được swap trước đó sẽ hiển thị `effectiveMode` đúng (WFH/WIO) thay vì OFF. Giao diện dùng dashed border nếu là weekend mà chưa có override.

3. **GitHub Actions trigger local** — `GITHUB_PAT` không có trong env local → GHA trigger bị skip (console.warn). Production trên Vercel thì OK.

4. **`app.js` vs `app-enhanced.js`** — `public/index.html` load `app-enhanced.js`. `app.js` là file cũ, giữ lại để reference. Không chỉnh sửa `app.js`.
